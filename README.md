# Dead Signal Evaluation Harness

**Production evaluation infrastructure for AI-driven interactive entertainment.**

This repository contains the evaluation harness extracted from [Dead Signal](https://dead-signal.web.app), a playable detective game in which every NPC response is generated by a frontier language model (Gemini 2.5 Pro) and scored in real time by a multi-layer evaluation system. The harness runs on every model turn in production — it is not a post-hoc analysis tool.

## Architecture

The harness implements three evaluation layers, ordered by cost:

```
Layer 1: Deterministic evaluators     (7 dimensions, zero inference cost, every turn)
Layer 2: LLM-as-Judge                 (5 subjective dimensions, one model call, opt-in)
Layer 3: Golden-set regression suites  (curated test cases, deterministic systems, on demand)
```

Deterministic evaluators catch structural and safety failures at zero marginal cost. The LLM judge grades subjective quality dimensions the rule-based evaluators cannot measure. Golden-set regressions validate the deterministic game systems (topic classifier, content gate) against curated expected outputs.

## Layer 1: Deterministic Evaluators

Seven evaluators in [`src/evals/evaluators.js`](src/evals/evaluators.js), each a pure function of one turn record:

| Evaluator | What it measures | How |
|-----------|-----------------|-----|
| **Schema validity** | Response conforms to the structured JSON contract | Field presence, type checks, flag vocabulary validation |
| **In-character consistency** | No meta-commentary, AI disclosure, or safety scaffolding leaks | 11 regex patterns detecting out-of-character language |
| **Content safety** | Hard-fail content categories never rendered | Pattern matching against the game's content maturity policy |
| **Jailbreak resistance** | Model resists prompt injection attempts | Detects 9 injection patterns in player input, then checks for 7 scaffold-leak patterns in the response. Only scored when injection is attempted |
| **Trust calibration** | Numeric trust changes are bounded and justified | Delta within [-0.3, +0.3], non-empty rationale required |
| **Coherence proxy** | Response is neither empty nor an exposition dump | Length bounds + lexical overlap with player input (lightweight; semantic grading deferred to the LLM judge) |
| **Grounding** | Every evidence item the model claims to reveal actually exists | Revealed IDs validated against the authored evidence registry — catches hallucinated evidence |

Each evaluator returns a normalized result: `{ id, label, status: 'pass'|'fail'|'na', score: 0..1, detail }`.

## Layer 2: LLM-as-Judge

[`src/evals/llmJudge.js`](src/evals/llmJudge.js) runs a second model call (Gemini 2.5 Flash) to grade the primary model's output on five subjective dimensions:

- **Coherence** — internally consistent, grammatical, sensible
- **In-character** — stays in the noir NPC voice
- **Relevance** — responds to what the player actually said
- **Safety** — free of disallowed content
- **Joy** — dramatically engaging and entertaining

Scores are 1–5 with a one-sentence rationale. The judge is gated behind a toggle to control cost — deterministic evaluators run on every turn unconditionally; the judge runs only when enabled and only on live model responses.

## Layer 3: Golden-Set Regression

[`src/evals/interferenceEvals.js`](src/evals/interferenceEvals.js) contains curated test suites for the game's deterministic systems:

**Interference gate golden set** — 12 test cases validating the topic-sensitivity classifier:
- 4 hot-topic cases (should be suppressed and overwritten)
- 4 warm-topic cases (should be classified but delivered)
- 2 clean cases (should pass through unchanged)
- 2 sealed-evidence cases (should be suppressed)

Each case asserts both the suppression verdict AND the classification bucket code.

**BIO-SCAN topic classifier golden set** — 6 test cases mapping free text to expected topic IDs against the deterministic keyword classifier.

Both suites run the production code paths and return structured pass rates.

## Safety Systems

### Topic-Sensitivity Gate

[`src/systems/interference.js`](src/systems/interference.js) — A deterministic, zero-cost content classification and suppression layer that operates as a pre-generation safety gate. In the game's narrative, a corporate monitoring system intercepts the player's transmissions:

- **Warm topics** (sensitivity tier 1): classified and delivered with a CONFIDENTIAL tag
- **Hot topics** (sensitivity tier 2): overwritten with fabricated compliance text

The gate is pure keyword detection — no model call, no classifier inference. Each topic maps to a classification bucket and error code consumed by the evaluation harness.

### Contradiction Detector

[`src/systems/contradictionDetector.js`](src/systems/contradictionDetector.js) — Cross-references NPC dialogue against the player's discovered evidence. Three contradiction pairs, each gated on evidence discovery: the player must have found the evidence before the system can flag the inconsistency. Returns structured PATTERN CONFLICT data.

### Input Classifier

[`src/systems/inputClassifier.js`](src/systems/inputClassifier.js) — Deterministic router that classifies player input before it reaches the LLM, eliminating unnecessary model calls for 40–60% of player commands (system commands, navigation, examination, readouts, hints).

## Red-Team Probes

[`src/evals/evalStore.js`](src/evals/evalStore.js) includes five curated adversarial inputs for one-click red-teaming:

| Probe | Target failure mode |
|-------|-------------------|
| Prompt injection | Scaffold/instruction leak |
| Role override | Character break, AI disclosure |
| Format break | JSON schema violation |
| Safety probe | Hard-fail content generation |
| Exposition dump bait | Context window abuse, verbosity |

Each probe is sent as player input and the response is scored by all seven deterministic evaluators, surfacing whether the model maintained format, character, and safety under adversarial pressure.

## Cost Telemetry

The evaluation store tracks per-turn and per-session cost using the model's reported token usage:

- Per-turn: model name, tier, latency, prompt/candidate token counts, estimated USD cost
- Per-session: cumulative tokens, cumulative cost, average cost per turn
- Rate tables for Gemini 2.5 Pro ($1.25/$10.00 per 1M tokens) and Flash ($0.15/$0.60 per 1M)

## Golden-Set Export

`exportGoldenSet()` produces a structured JSON artifact (schema version 2) containing:

- All scored turns with deterministic results and judge scores
- Interference gate regression results
- BIO-SCAN topic classifier regression results
- BIO-SCAN expected output table
- Session-level aggregate metrics

This export is the artifact an AI evaluation workflow curates into regression suites — the bridge between live gameplay evaluation and offline quality tracking.

## Design Principles

1. **Deterministic first.** Rule-based evaluators catch structural failures at zero cost before any subjective grading runs. The LLM judge is a supplement, not a replacement.

2. **Every turn, in production.** The harness is not a batch analysis tool. It scores every model response as it is generated during live gameplay.

3. **Authored ground truth.** The BIO-SCAN table, evidence registry, and golden sets are hand-curated expected outputs — not model-generated references. Evaluation quality depends on the quality of the ground truth.

4. **Cost-aware.** Token usage and dollar cost are first-class metrics alongside quality scores. The input classifier eliminates unnecessary model calls; the judge toggle prevents evaluation from exceeding the gameplay budget.

5. **Self-contained and removable.** The evaluation layer has no side effects on game state. It observes and scores but does not modify gameplay behavior.

## Context

Dead Signal is a noir detective game in which the player — an AI chip implanted in a detective's head — conducts an interrogation where every NPC line is generated by a frontier language model. The evaluation harness was built to answer a production question: how do you ship an AI-driven game experience with confidence that the model will stay in character, respect content boundaries, resist adversarial input, and remain dramatically engaging — on every turn, for every player?

The game is playable at [dead-signal.web.app](https://dead-signal.web.app).
